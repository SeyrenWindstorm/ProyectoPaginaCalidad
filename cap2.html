CAPITULO II. ENTORNO DE LOS PRODUCTO DE SOFTWARE


1. METODOLOGIA DE DESARROLLO DE SISTEMAS APLICADA

a. MODELO DE CICLO DE VIDA APLICADO


Se aplicará la metodología SCRUM.

Planificación del Sprint:
  • Seleccionar elementos del Backlog del Producto para abordar en el próximo Sprint.
  • Estimar la cantidad de trabajo que se puede realizar en el Sprint.
Sprint:
  • Desarrollar y probar las funcionalidades seleccionadas durante la Planificación del Sprint.
  • Realizar reuniones diarias (Daily Scrum) para actualizar el progreso y abordar cualquier impedimento.
Revisión del Sprint:
  • Mostrar y demostrar el trabajo completado al final del Sprint.
  • Obtener comentarios del cliente y partes interesadas.
Retrospectiva del Sprint:
  • Reflexionar sobre el Sprint pasado y las prácticas de trabajo.
  • Identificar oportunidades de mejora y definir acciones para implementar en el próximo Sprint.

2. REQUISITOS DE APLICACIÓN

a. REQUISITOS FUNCIONALES


El sistema debe permitir a los usuarios crear, editar, compartir y consultar documentos, imágenes, vídeos y otros tipos de contenido relacionados con el conocimiento de la organización.

El sistema debe permitir a los usuarios organizar el contenido en bibliotecas, carpetas, etiquetas y metadatos, para facilitar su búsqueda y acceso.

El sistema debe permitir a los usuarios crear y participar en comunidades de práctica, foros de discusión, blogs y wikis, para fomentar la colaboración y el intercambio de conocimiento.

El sistema debe permitir a los usuarios crear y gestionar flujos de trabajo, tareas y alertas, para automatizar y controlar los procesos relacionados con el conocimiento

El sistema debe permitir a los usuarios personalizar la interfaz y las funcionalidades del sistema, según sus preferencias y roles.


b. REQUISITOS NO FUNCIONALES


El sistema debe garantizar la seguridad y la privacidad de los datos, mediante el uso de protocolos de cifrado, autenticación y autorización.

El sistema debe garantizar la disponibilidad y la accesibilidad del contenido, mediante el uso de servicios en la nube, sincronización y copias de seguridad.

El sistema debe garantizar la escalabilidad y la adaptabilidad del sistema, mediante el uso de arquitecturas modulares, flexibles y personalizables.

El sistema debe garantizar la usabilidad y la satisfacción del usuario, mediante el uso de diseños intuitivos, interactivos y responsivos.


c. ESPECIFICACIÓN DE REQUERIMIENTOS DE SOFTWARE Y SUS ATRIBUTOS DE CALIDAD.


La ERP tiene como objetivo establecer un contrato entre el cliente y el proveedor, definir el alcance y los límites del proyecto, facilitar la comunicación y el entendimiento entre las partes, y servir de base para el diseño, la implementación, la prueba y la entrega del sistema.

La ERP debe seguir una estructura y un formato estándar, que facilite su lectura, comprensión y verificación.
